Stop Automatic Lot Number Allocation Odoo App

Stop Auto Lot Allocation-Selection on Delivery Order and picking odoo app useful for manual lot number selection on delivery order and internal transfer. By Default on Odoo Lot number will be automatically allocated on delivery order and transfer based on removal strategy when picking become available by check availability button or with confirm sales order, So the problem is user don't have choice to select lot number manually on delivery order and picking. After installing our stop auto lot selection odoo app user need to enable stop auto lot number option and automatic reservation on lot is stopped, So user have choice to select specific lot on delivery order based on customer requirement.

Features

Stop Auto Assign Lot from Sales.

Stop auto assign lot number on picking from sales order.

Stop Auto Assign Lot from Inventory Scheduler.

Stop auto assign lot number on picking from running inventory scheduler.

Default Odoo Flow

With default odoo Lot number selection on delivery order works based on removal strategy configured on product category, for example FIFO, FEFO, LIFO.

If you don't select removal strategy then it will takes by default FIFO, So first created lot automatically reserve when sale order is confirmed and stock is available for that specific product.

So the problem is user don't have choice to select lot on delivery order because lot number automatically selected and user don't have choice to select lot number manually.

Sales Order

User can see validated sales order with 30 ordered quantity.

Delivery Order

User can see delivery order in "Ready" state and lots/serial numbers automatically assigned as per ordered quantity.

On detailed operation wizard user can see auto assigned lot number for product as per ordered quantity.

Solution

After install our module automatic reservation on lot is stopped, So user have choice to select specific lot on delivery order based on customer requirement.

With this odoo app we provide a solution that when sales order is confirmed delivery order is not reserve lot number automatically, So warehouse user open delivery order and select required lot and deliver to the customer.

User can see now module is installed.

Stop Auto Lot Allocation Configuration

UNder Inventory > Configuration > Settings menu user need to enable "Stop auto lot number" option to stop automatic lot number allocation.

Sales Order

User can see validated sales order with 27 ordered quantity.

Delivery Order

User can see delivery order in "Waiting" state and lots/serial numbers are not assigned automatically.

On detailed operations wizard user can see lot number are not assigned automatically.

User have to select lot/serial number and done quantity manually for product as per ordered quantity and click on "Confirm" button.

On click of confirm button user can see delivery order state is changed to "Ready" from "Waiting".

User can also on validate delivery order by clicking on "Validate" button, delivery order state will changed to "Done" from "Ready".
